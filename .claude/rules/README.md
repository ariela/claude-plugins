# ClaudeCode ルールとメモリシステム

このディレクトリには、ClaudeCodeがプロジェクトで作業する際に参照すべきルール、ガイドライン、およびプロジェクト知識が格納されています。

```
.claude/rules/
├── README.md                      # このファイル - 全体概要
├── document-update-workflow.md    # ドキュメント更新のワークフロー
├── coding-standards.md            # コーディング規約とベストプラクティス
├── troubleshooting.md             # トラブルシューティングガイド

└── project-memory.md              # プロジェクト固有の知識とメモリ
```

## 🎯 使用目的

### ClaudeCodeへの指示

作業を開始する前に、必ず以下のファイルを確認してください：

1. **document-update-workflow.md** - ドキュメント更新の手順とルール
2. **coding-standards.md** - コーディング規約
3. **troubleshooting.md** - 既知の問題と解決策
4. **patterns.md** - 推奨される実装パターン
5. **project-memory.md** - プロジェクト固有の重要な情報

## 📝 ドキュメント更新の原則

### 更新が必要なタイミング

以下の状況では、必ずドキュメント更新を提案してください：

1. ✅ エラーや問題を解決した時
2. ✅ 効率的な実装パターンを発見した時
3. ✅ 新しいAPI/ライブラリの使用方法を確立した時
4. ✅ 既存ドキュメントの情報が古い/不正確だと判明した時
5. ✅ 頻繁に参照される情報を発見した時
6. ✅ コードレビューの修正を完了した時

### 更新プロセス

```
1. 新しい知識を発見
   ↓
2. 💡 ユーザーに更新提案を提示
   ↓
3. ユーザーが承認・編集・却下を選択
   ↓
4. 承認された場合のみドキュメントを更新
   ↓
5. 更新日時を記録
```

## 🔒 重要な制約

- **絶対にユーザーの承認なしにドキュメントを更新しない**
- 機密情報（APIキー、パスワード等）は記録しない
- 既存の記載スタイルとフォーマットを踏襲する
- 更新日時（YYYY-MM-DD形式）を必ず記録する

## 📚 ファイル分割のガイドライン

単一ファイルが100行を超えた場合、以下の基準で分割を検討してください：

- **トピック別**: 関連する内容をグループ化
- **頻度別**: よく参照される内容は独立ファイルに
- **カテゴリ別**: エラー/パターン/API使用法など
